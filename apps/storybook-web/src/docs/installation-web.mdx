# Installation (Web)

How to integrate the Swiss Activities Design System into a web project using a git submodule.

## Prerequisites

- **git** — the DS is consumed as a git submodule
- **Tailwind CSS** — v4 recommended, v3 supported (see legacy section below)

## 1. Add the DS as a git submodule

```bash
git submodule add https://github.com/Swiss-Activities/ds.git ds
```

After cloning a repo that already has the submodule:

```bash
git submodule update --init --recursive
```

## 2. Add the dependency

Add a `file:` dependency in your `package.json` pointing to the UI package inside the submodule:

```json
{
  "dependencies": {
    "@swiss-activities/ui": "file:./ds/packages/ui"
  }
}
```

Then run your package manager:

```bash
bun install
# or
yarn install
```

## 3. Configure TypeScript paths

Add path aliases in `tsconfig.json` so TypeScript resolves bare `@swiss-activities/ui` imports to the local source:

```json
{
  "compilerOptions": {
    "paths": {
      "@swiss-activities/ui": ["ds/packages/ui/src/index.ts"],
      "@swiss-activities/ui/*": ["ds/packages/ui/src/*"]
    }
  }
}
```

## 4. Configure Tailwind CSS

### Tailwind v4 (recommended)

Install the Vite plugin or PostCSS plugin for Tailwind v4. Then in your global CSS file:

```css
@import "tailwindcss/theme.css" layer(theme);
@import "@swiss-activities/ui/styles/sa-theme.css";
@import "tailwindcss/preflight.css" layer(base);
@import "tailwindcss/utilities.css" layer(utilities);

/* Tell Tailwind to scan DS source files for class names */
@source "../../../ds/packages/ui/src/**/*.{ts,tsx}";
```

The theme import provides design tokens (colors, spacing, etc.) as CSS custom properties.

### Tailwind v3 (legacy)

The **website** project currently uses Tailwind v3. This requires a classic config-based setup.

**`postcss.config.js`**

```js
module.exports = {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
};
```

**`tailwind.config.ts`** — import tokens from the DS and add the DS source to `content`:

```ts
import type { Config } from "tailwindcss";
import { grayColors, saColors } from "@swiss-activities/ui/tokens/colors";
import { saKeyframes, saAnimations } from "@swiss-activities/ui/tokens";
import { screens } from "@swiss-activities/ui/tokens/screens";

const config: Config = {
  content: [
    "./src/**/*.{js,ts,jsx,tsx}",
    "./ds/packages/ui/src/**/*.{js,ts,jsx,tsx}",
  ],
  theme: {
    screens,
    extend: {
      colors: {
        gray: grayColors,
        dark: saColors.dark,
        primary: saColors.primary,
        medium: saColors.medium,
        light: saColors.light,
        blue: saColors.blue,
        bg: saColors.bg,
      },
      keyframes: saKeyframes,
      animation: saAnimations,
    },
  },
};

export default config;
```

**Global CSS** — import the theme CSS:

```css
@import "@swiss-activities/ui/styles/sa-theme.css";
```

## 5. Framework-specific setup

### Vite

Works out of the box. Vite resolves `file:` dependencies and TypeScript paths natively.

### Next.js

Next.js uses webpack/turbopack, so you need to tell it to transpile the DS package. In `next.config.js`:

```js
const nextConfig = {
  transpilePackages: ["@swiss-activities/ui"],
};
```

TypeScript path aliases from `tsconfig.json` are picked up automatically by Next.js.

## Usage

### Components

```tsx
import { Button } from '@swiss-activities/ui'

export function Example() {
  return <Button variant="primary" size="md">Book now</Button>
}
```

### Tokens

```ts
import { saColors } from '@swiss-activities/ui/tokens/colors'
import { screens } from '@swiss-activities/ui/tokens/screens'
import { saKeyframes, saAnimations } from '@swiss-activities/ui/tokens'
```

### Types

```ts
import type { ButtonProps } from '@swiss-activities/ui'
```

## Current integrations

- **website** — Next.js + Tailwind v3 (legacy) + git submodule
- **mobile-tickets** — Vite + Tailwind v4 + git submodule
